06 both
App icon finalization
iOS needs 1024×1024 · Android needs adaptive icon layers · generate from existing assets
2–3 hrs
by May 15
iOS requirements
Single 1024×1024px PNG. No transparency. No rounded corners (Apple applies the mask). Must be legible at 29×29px. Generate all sizes from this master using Xcode's asset catalog or makeappicon.com.
Android requirements
Adaptive icon = foreground layer (icon art, 108×108dp safe zone = center 72dp) + background layer (solid color or pattern). Bubblewrap will generate these from your manifest icons if they're correctly sized. Your existing icon-512.png and icon-512-maskable.png are the right starting point.
acceptance criteria
  • 1024×1024 PNG exported, no transparency, no alpha
  • Icon reads clearly at thumbnail size (60×60px)
  • Matches E15 brand palette (ink bg, voice accent)
  • Added to Xcode asset catalog (iOS)
  • icon-512-maskable.png confirmed safe-zone-compliant (Android)

Xcode WKWebView wrapper
Thin native SwiftUI shell · loads the Cloudflare Pages URL · no app rewrite needed
1 day
by May 20
what to do
Open Xcode → New Project → App (SwiftUI). Set bundle identifier (e.g. com.e15solutions.eversmenu). In ContentView, replace the default body with a WKWebView loading your Cloudflare Pages URL. Add WKWebView via UIViewRepresentable wrapper. Enable JavaScript. Set background to black. That's the whole app.
// ContentView.swift (simplified) import SwiftUI import WebKit struct WebView: UIViewRepresentable { let url = URL(string: "https://evers-menu.pages.dev")! func makeUIView(context: Context) -> WKWebView { let config = WKWebViewConfiguration() config.allowsInlineMediaPlayback = true let wv = WKWebView(frame: .zero, configuration: config) wv.backgroundColor = UIColor(red:0.04,green:0.04,blue:0.04,alpha:1) wv.load(URLRequest(url: url)) return wv } func updateUIView(_ uiView: WKWebView, context: Context) {} } struct ContentView: View { var body: some View { WebView().ignoresSafeArea() } }

10 android ⚠ gate
Digital Asset Links (assetlinks.json)
Proves the Play Store app owns the domain · without this TWA falls back to Chrome UI
30 min
by May 17
⚠ gating — without this file correctly deployed, the TWA shows Chrome's browser UI instead of running full-screen. Kills the native feel.
what to do
Bubblewrap generates the contents of this file during init. After running bubblewrap init, copy the generated SHA-256 fingerprint and create the file at /.well-known/assetlinks.json in your Cloudflare Pages deploy. Must be served with Content-Type: application/json.
// /.well-known/assetlinks.json (Bubblewrap generates this for you) [{ "relation": ["delegate_permission/common.handle_all_urls"], "target": { "namespace": "android_app", "package_name": "com.e15solutions.eversmenu", "sha256_cert_fingerprints": ["YOUR_FINGERPRINT_FROM_BUBBLEWRAP"] } }]
acceptance criteria
  • assetlinks.json live at https://[your-domain]/.well-known/assetlinks.json
  • File returns correct Content-Type: application/json
  • SHA-256 fingerprint matches the keystore from Bubblewrap
  • Verified via: https://digitalassetlinks.googleapis.com/v1/statements:list?source.web.site=https://[your-domain]&relation=delegate_permission/common.handle_all_urls
